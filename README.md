# PackUp 이사 서비스 웹사이트

디자이너가 제공한 Figma UI 시안을 기반으로  
웹 환경에 맞게 퍼블리싱 및 프론트엔드 구현을 진행한  
이사 서비스 소개 웹사이트입니다.
모바일 환경에서도 빠르고 직관적으로 정보를 확인할 수 있도록 UI와 인터랙션에 중점을 두어 제작했습니다.

실제 협업 상황을 가정해,
디자이너가 전달한 시안을 해석하고
웹 환경에서 동일한 경험을 제공하는 데 집중했습니다.

---

## 프로젝트 개요

- 프로젝트 형태: 개인 프로젝트
- 제작 목적:  
  이사 서비스 웹사이트를 가정한 UI 구현을 통해  
  반응형 레이아웃, 인터랙션, 접근성까지 고려한 프론트엔드 구현 경험을 쌓기 위함
- 제작 기간: 약 2주
- 참여 인원: 1명
- 기여도:
  디자이너 제공 Figma 시안 분석부터  
  퍼블리싱, 인터랙션 및 반응형 프론트엔드 구현까지 100% 담당


---

## 기술 스택

- HTML5
- CSS3
- JavaScript
- Git / GitHub

### 기술 선택 이유
- 라이브러리 없이 DOM 구조와 이벤트 흐름을 이해하기 위해 JavaScript 사용
- 복잡한 레이아웃을 안정적으로 유지하기 위해 CSS Grid 기반으로 전체 레이아웃 설계
- 다양한 디바이스 환경을 고려해 미디어 쿼리 중심의 반응형 구현

---

## 레이아웃 설계 방식

- 초기 화면 구현 과정에서 레이아웃 구조를 한 번에 잡는 것이 쉽지 않아,
눈에 보이는 레이아웃 가이드를 직접 만들어 화면 구조를 설계했습니다.

```html
<div class="layout-guide">
  <div></div>
  <div></div>
  <div></div>
  <div></div>
  <div></div>
</div>
```


- 해당 가이드는 실제 서비스에는 노출되지 않으며,
각 영역의 비율과 배치를 시각적으로 확인하기 위한 용도로 사용했습니다.
레이아웃 가이드를 기준으로 콘텐츠를 하나씩 배치하며
전체 화면 구조를 점검한 후, 최종 구현 단계에서 제거했습니다.

- 이 과정을 통해 화면 구조를 명확히 인지할 수 있었고
반응형 전환 시 레이아웃이 무너지는 문제를 줄일 수 있었으며
디자이너 시안과 웹 구현 간의 오차를 최소화할 수 있었습니다.

## 주요 기능

- Desktop / Tablet / Mobile 반응형 레이아웃
- 모바일 햄버거 메뉴 토글 UI

- FAQ 섹션 아코디언 UI (details / summary)

## 문제 해결 경험

### 1. 모바일 화면에서 레이아웃이 깨지는 문제 발생

**문제**
데스크탑 기준으로 설계된 네비게이션이 모바일 화면에서 요소간 간격이 깨지고, 메뉴 접근성이 떨어지는 문제가 발생했다.

**원인**
- 화면 크기 변화에 따라 요소가 단순히 축소됨
- 모바일 환경에 적합한 UI 구조가 유지되지 않음

**해결**
- 미디어 쿼리를 활용해 모바일 브레이크포인트를 설정
- 네비게이션의 배치 방식과 인터랙션 흐름 분리
- 화면 크기에 따라 구조가 전환되는 반응형 레이아웃으로 재설계
---

### 2. 메뉴 토글 시 레이아웃 밀림 현상

**문제**
- 네비게이션 토글 시 메뉴가 열리고 닫히는 과정에서 페이지 전체 레이아웃이 흔들리거나 위치가 달라지는 문제 발생

**원인**
- 토글 메뉴 오픈 시 배경 스크롤이 함께 움직이며 레이아웃이 흔들리는 현상이 있음

**해결**
- 메뉴가 열릴 때 body에 클래스를 추가해 스크롤을 막고, 닫힐 때 해제
---

## 배운 점

- 반응형 레이아웃은 화면을 줄였을 때 요소를 단순히 축소하는 것이 아니라, 디바이스 환경에 맞게 레이아웃 구조 자체를 재설계하는 과정이라는 것을 배움
- 모바일 환경에서는 네비게이션의 배치 뿐만 아니라 토글 방식, 레이어 구조, 화면 흐름까지 함께 설계해야 안정적인 UI 를 구현할 수 있다
- 초기에는 빠르게 구현하는 데 집중했지만, CSS 구조를 분리하지 않으면 수정 시 영향 범위가 커진다는 것을 느꼈고, 이후 공통 스타일과 페이지 스타일을 구분하려는 기준이 생겼다.
